SAP Sailing Analytics Webservices API Version 1.0

URL: /api/v1/scopemigration/movetodedicatedserver

Description:

This endpoint is used to carve out a group of leaderboard groups from a server and move them to a dedicated, possibly new server that contains only the selected leaderboard groups. The result of the move is then compared to the original, and a remote reference from the exporting to the new dedicated server is established. Should any of these steps produce a non-OK response code, the overall response code will be that, and the responseCode fields of the individual objects reported in the response will show the repsonse of each sub-step.


Webservice Type: POST
Output format: json
Mandatory parameters:
server2, the host name of the server where the groups should be moved to.
leaderboardgroupUUID[], a set of leaderboardgroup UUIDs.
user1, username for server1.
user2, username for server2.
password1, password for user1 on server1.
password2, password for user2 on server2.
bearer1, bearer token for user1 on server1, to provide instead of username and password.
bearer2, bearer token for user2 on server2, see bearer1.
override, tells whether existing content with equal identify on the importing side shall be overwritten by this action; defaults to "false"
Optional parameters:
server1, the host name of the server from where you want to "export" the leaderboardgroups. Defaults to the server processing this request.
user1, username for archive server. Provide password1 with it. Alternatively, specify bearer1 or default to the authentication of this request.
user2, username for dedicated server. Provide password2 with it. Alternatively, specify bearer2 or default to the authentication of this request.
password1, password for user1 on archive server.
password2, password for user2 on dedicated server.
bearer1, bearer token to authenticate a user on the archive server, to provide instead of username1 and password1.
bearer2, bearer token to authenticate a user on the dedicated server, to provide instead of username2 and password2.
override, tells whether existing content with equal identify on the importing side shall be overwritten by this action; defaults to "false"
Request method: POST
Example: curl -X POST localhost:8889/sailingserver/api/v1/scopemigration/movetodedicatedserver -d "user1=admin" -d "user2=admin" -d "password2=admin" -d "password1=admin" -d "leaderboardgroupUUID[]=75e86618-8b0f-4291-9661-6b2a1fa47dff" -d "server1=localhost:8888" -d "server2=localhost:8889" -H "Authorization: Basic YWRtaW46YWRtaW4="
