SAP Sponsorships Engine Security Webservices API

URL: /api/restsecurity/role

Description:

The service reads and updates a role and the associated permissions based on the HTTP method used.


Request Method: GET
Output format: a JSON document of the following format
{"permissions": ["BOAT:READ", "COMPETITOR,EVENT:READ"], "id": "29914406-b0fc-43cd-acc7-62a83931f587", "name": "test-role"}
Mandatory parameters:
roleId: UUID of the role to get.
Example: curl -X GET "127.0.0.1:8888/security/api/restsecurity/role?roleId=29914406-b0fc-43cd-acc7-62a83931f587"
Will provide a JSON document as explained above.
Request Method: POST
Output format: An empty response with status code 200 in case of success; error-indicating status codes otherwise
Mandatory parameters:
roleId: UUID of the role to update.
permissions: Set of String permissions, the role's current permissions will be replaced with.
Example: curl -X POST "127.0.0.1:8888/security/api/restsecurity/role?roleId=29914406-b0fc-43cd-acc7-62a83931f587&permissions=BOAT:READ&permissions=COMPETITOR,EVENT:READ"
